★ HOW IT WORKS

Up and running in four steps

1
INSTALL

Install Python 3.11+ and dependencies

Clone the repo, create a virtual environment, run pip3 install -r requirements.txt and playwright install chromium. Full instructions in START_HERE.md for both Windows and Mac.

2
CONFIGURE

Run the setup wizard

Run python3 setup.py — it asks for your Dice credentials, target job type, keywords, and daily speed. Configures everything automatically in under 3 minutes.

3
RUN

Start the bot

Run python3 dice_bot.py. A Chrome window opens, logs in, and starts applying. Keep the terminal open or deploy to Oracle Cloud free tier for 24/7 headless operation.

4
RECEIVE

Answer inbound vendor calls

Your profile is now inside thousands of vendor ATS systems. When matching roles open, vendors find you and call. Check applications.csv anytime to see the full log.

★ FAQ

Common questions

⚠ Does this violate Dice.com's Terms of Service? ▾
Yes. Automated job applications are against Dice's ToS. The bot includes anti-detection measures — Bezier mouse paths, human typing delays, fingerprint spoofing — but no tool guarantees zero account risk. Use at your own discretion.
Which jobs does it apply to? ▾
Easy Apply jobs only — the ones with a one-click apply button hosted directly on Dice. Jobs that redirect to external ATS platforms (Workday, Greenhouse, Lever) are automatically skipped and closed.
Do I need to upload a resume file? ▾
No. DiceBot uses your resume that's already uploaded to your Dice.com account. No local resume file needed — just your login credentials.
How many applications per day is safe? ▾
250–300/day is the tested safe rate — proven over a real 10-day run with 0 detections. Running above 300/day increases risk. Control via BATCH_SIZE and BATCH_BREAK_MINUTES in config.py.
Can I run it 24/7 for free? ▾
Yes. Oracle Cloud Free Tier gives you a permanently free ARM VM (4 OCPUs, 24GB RAM). Set HEADLESS=True in config.py, deploy there, and it runs unattended. Full instructions in README.md.
What if it gets logged out mid-run? ▾
DiceBot automatically detects login page redirects anywhere during the run and re-authenticates using your stored credentials. It then continues exactly where it left off.
